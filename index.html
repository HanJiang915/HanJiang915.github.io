Project - Geospatial Analysis for Disaster Planning

I am working with three other students who major in computer science now for this project "Geospatial Analysis of Disaster Planning". We are partnered with Dr. Joseph Louis, who is a civil engineering professor from Oregon State University.

This project is intended to provide emergency managers with a systems level view of the impact of disasters on their jurisdictionâ€™s lifeline networks. Our idea is creating a tool that can help emergency managers to make robust response plan by identifying critical intersections of lifeline networks that may cause bottlenecks in recovery. Our tool aim to provide emergency managers a immediate response plans by identifying critical intersections of lifeline networks that may casue bottlenecks in recovery. It will also calculate the estimated time for them to arrive the destination and predict resources needed for clearing the devris.

The software we use for this project is ArcGIS pro and HAZUS. The tools for developing are ArcGIS built-in function toolbox and national public database. We write python scripts to implement the toolboxes. And we use database and HAZUS to make simulations of different disasters and create necessary layers for ArcGIS. For using our tool within ArcGIS, once input the area to clear, the debris raster, and other information about the emergency management team, emergency managers can get the estimated clearance time.

Our team created three scrips for the tool.

First is random raster generation script. We need to use raster data to do the calculation. We originally wanted to find existed raster data to simulate the spread of debris when diaster occuring in Astoria, OR. However, there are no available earthquake for us to use after searching online. So, we wrote a script to simulate debris spread to run the other scripts with raster data.

Second is a clearing time estimation script. This script use the previous stimulated raster data with the data of a clearance area to calculate the needed time for cleaning the debris on that area. Dr. Louis gave us equations for reference to create the calculations.

Third is for calculating the debris volume. The difficult we met when creating this script was that the stimulated raster data would not proide us the exact volume of debris. We then find a way to calculate certain cells and then get the output of the volume of debris.
